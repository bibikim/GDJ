* Redirect ( <a>와 location은 redirect 방식)

 - 1차요청 -> 서버1에서 작업 완료 > 완료 후 이동할 장소를 알려줌 > 그 알림을 받고 다시 서버2로 이동


 -  1차 요청 - insert 요청 -> 서버1에서 insert 완료
    서버1이 목록보기(select)로 이동하라고 주소를 알려줌(=>redirect)
    그 주소를 가지고 목록보기(select)로 이동하며 서버2로 넘어가서 새로운 요청(검색)

 - redirect는 두 번의 요청이 일어난다
   첫번째 요청과 두번째 요청은 다른 요청이다.
   기존의 요청 정보는 유지되지 않고(사라지고) 이동된다.



* forward

 - 요청을 a 했는데 그 요청이 b로 그대로 전달해주는 것
   요청은 a에 했지만 마무리 작업은 b에서 응답

 - forward ex)
   select 요청 -> 이 요청에 대한 결과를 곧바로 response해주는게 아니라 해당 결과를 request에 실어서 제2의 서버(응답할 경로)로 전송시켜주고 보여줌.

 - 서버 내부에서 단순 이동할 때 사용함


  ◈  요청을 보내주지 않는건 redirect (기존의 요청을 전달X)
       요청을 보내주는 건 forward (기존의 요청 전달O)

---------------------------------------------- 
insert update delete -> Redirect
select -> Forward
